# udpmulticast_sink 发送报文说明

本文档用于指导接收方如何接收、解析 `udpmulticast_sink` 发送的 UDP 组播报文。

## 1. 传输方式

- 传输协议：UDP Multicast
- 单包格式：一个 UDP Datagram 对应一个完整 JSON 字符串
- 编码方式：UTF-8 文本
- 无额外二进制头、无长度前缀、无压缩

也就是说，接收端从 socket `recvfrom()` 拿到的数据，直接按 JSON 解析即可。

## 2. 报文发送行为

当前插件的发送行为如下：

- 每次发送 1 个完整 JSON 报文
- 每个报文对应 1 路视频源的 1 次发送周期
- 同一个报文内的 `cont` 数组包含该路当前帧的所有目标
- 如果该帧没有检测到目标，也会发送 1 个占位目标，`trk_stat=0`，`tar_iden="none"`
- 多路视频场景下，会按 `source_id` 分别发送

说明：

- `source_id` 是区分不同视频源的关键字段
- `msg_sn` 是发送计数，按插件实例递增，不保证跨多个 sink 实例全局唯一

## 3. 顶层 JSON 结构

报文顶层结构如下：

```json

  "msg_sn": 12,
  "msg_type": 3,
  "tx_sys_id": 0,
  "tx_dev_type": 1,
  "tx_dev_id": 0,
  "tx_subdev_id": 0,
  "rx_sys_id": 0,
  "rx_dev_type": 1,
  "rx_dev_id": 0,
  "rx_subdev_id": 0,
  "yr": 2026,
  "mo": 3,
  "dy": 24,
  "h": 10,
  "min": 49,
  "sec": 17,
  "msec": 123.456,
  "cont_type": 1,
  "cont_sum": 2,
  "cont": [
    {
      "yr": 2026,
      "mo": 3,
      "dy": 24,
      "h": 10,
      "min": 49,
      "sec": 17,
      "msec": 123.456,
      "dev_id": 0,
      "guid_id": 0,
      "tar_id": 0,
      "trk_stat": 1,
      "trk_mod": 0,
      "fov_angle": 0.0,
      "lon": 0.0,
      "lat": 0.0,
      "alt": 0.0,
      "tar_a": 0.0,
      "tar_e": 0.0,
      "tar_rng": 0.0,
      "tar_av": 0.0,
      "tar_ev": 0.0,
      "tar_rv": 0.0,

```

说明：

- `msg_id=28946` 即十六进制 `0x7112`
- `cont_sum` 表示 `cont` 数组中的目标数量
- 接收方应以 `cont` 为目标列表，不要自己假设固定目标数

## 4. 顶层字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `msg_id` | int | 固定为 `0x7112`，十进制显示时通常为 `28946` |
| `msg_sn` | int | 报文计数，按发送顺序递增 |
| `msg_type` | int | 固定为 `3`，表示数据流 |
| `tx_sys_id` | int | 固定为 `0` |
| `tx_dev_type` | int | 固定为 `1` |
| `tx_dev_id` | int | 固定为 `0` |
| `tx_subdev_id` | int | 固定为 `0` |
| `rx_sys_id` | int | 固定为 `0` |
| `rx_dev_type` | int | 固定为 `1` |
| `rx_dev_id` | int | 固定为 `0` |
| `rx_subdev_id` | int | 固定为 `0` |
| `yr` `mo` `dy` `h` `min` `sec` `msec` | int/float | 报文头时间戳 |
| `cont_type` | int | 固定为 `1`，表示多目标 |
| `cont_sum` | int | `cont` 数组长度 |
| `cont` | array | 目标数组 |

## 5. `cont` 目标字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `yr` `mo` `dy` `h` `min` `sec` `msec` | int/float | 目标时间戳 |
| `dev_id` | int | 设备类型，当前固定为 `0` |
| `guid_id` | int | 当前固定为 `0` |
| `tar_id` | int | 当前固定为 `0` |
| `trk_stat` | int | `1` 正常，`0` 丢失，占位目标通常为 `0`，`2` 外推 |
| `trk_mod` | int | 当前固定为 `0` |
| `fov_angle` | double | 当前固定为 `0.0` |
| `lon` `lat` `alt` | double | 当前固定为 `0.0` |
| `tar_a` `tar_e` `tar_rng` | double | 当前固定为 `0.0` |
| `tar_av` `tar_ev` `tar_rv` | double | 当前固定为 `0.0` |
| `tar_category` | int | 目标类别编码，当前固定为 `9` |
| `tar_iden` | string | 目标标签，当前固定为 `uav`；占位目标仍为 `none` |
| `tar_cfid` | float | 置信度 |
| `fov_h` `fov_v` | double | 当前固定为 `0.0` |
| `offset_h` `offset_v` | int | 当前固定为 `0` |
| `tar_rect` | int | 目标位置相关值，当前实现为目标框中心点 X 像素坐标 |
| `source_id` | int | 视频源编号，接收方区分多路视频的关键字段 |

## 6. 当前类别映射

插件当前的目标类别映射规则如下，当前仅保留一个有效类别 `uav`：

| 输入类别 | 输出 `tar_category` | 输出 `tar_iden` |
|----------|---------------------|-----------------|
| `0` | `9` | `uav` |

如果没有检测到目标，则发送占位目标：

- `trk_stat=0`
- `tar_category=0`
- `tar_iden="none"`
- `tar_cfid=0.0`
- `tar_rect=0`

## 7. 接收端最小实现

### Python 示例

```python
import json
import socket
import struct

GROUP = "230.1.8.31"
PORT = 8128
IFACE_IP = "192.168.1.114"  # 改成接收机本机网卡 IP

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM, socket.IPPROTO_UDP)
sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
sock.bind(("", PORT))

mreq = socket.inet_aton(GROUP) + socket.inet_aton(IFACE_IP)
sock.setsockopt(socket.IPPROTO_IP, socket.IP_ADD_MEMBERSHIP, mreq)

while True:
    data, addr = sock.recvfrom(65535)
    msg = json.loads(data.decode("utf-8"))
    for target in msg.get("cont", []):
        print(
            "source_id=", target.get("source_id"),
            "tar_iden=", target.get("tar_iden"),
            "tar_cfid=", target.get("tar_cfid"),
            "trk_stat=", target.get("trk_stat"),
        )
```

## 8. 接收方处理建议

建议接收方按以下逻辑处理：

1. 先按 UDP 组播收包
2. 每个 UDP 包直接按 UTF-8 JSON 解析
3. 校验 `msg_id == 0x7112`
4. 遍历 `cont` 数组
5. 按 `source_id` 对不同视频源分别处理
6. 如果 `trk_stat == 0`，按“该路当前无目标”处理

注意：

- 不要假设 `msg_sn` 在多路场景下一定全局连续
- 不要假设每包只有 1 个目标
- 不要假设每包一定有检测结果，占位目标是合法报文
